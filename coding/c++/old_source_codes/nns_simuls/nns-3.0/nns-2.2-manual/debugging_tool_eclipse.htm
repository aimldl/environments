5.1. Overview

Eclipse is an IDE (Integrated Development Environment) that works both on Windows and Linux. To develop the simulator, Eclipse is used mainly for debugging purposes.DDD was used  initially, a popular GUI debuggerin the Linux domain, but switched to Eclipse for its convenience. Eclipse is located in a sub-directory eclipse.

5.2. How to set up Workspace

This step is very important as failure to set up the correct Workspace may result in wrong operation of nns.

Step 1. Launch Eclipse.
Step 2. Select the "File" menu, "Switch Workspace", and "Other". Then "Workspace Launcher" window will pop up.
Step 3. Click the "Browse" button in "Workspace" and then select the project directory. For example, "/home/username/nns-all-in-one-1.0/nns-1.0".
Step 4. Click the "OK" button. Eclipse may restart automaticallly.

Note a sub-directory will be created in the directory specified as workspace when a project is created in Eclipse. Import the nns-1.2_dbg.exe in the name of "eclipse_project'".
Then a sub-directory with the same name will be created under the project directory. In other words, "nns-1.2/eclipse_project'".

5.3. How to create a C/C++ project

There are three main steps to set up Eclipse for the first time. First, create a project by importing the C++ executable nns1-2_dbg.exe. Secondly, configure the project by setting "Arguments" and "Source".Lastly,

Step 1. Launch Eclipse.
Step 2. Create a project by importing nns1-2_dbg.exe.
          . Click "Import" in the "File" Menu. Then "Import" Window will pop up.
          . Select C/C++ - C/C++ Executable. Click the "Next" button.
          . Now, the new step is to set "Import C/C++ Executable Files".
            Click the "Browse button for "Select executable". Find and select
            "nns1-2_dbg.exe" in directory bin under the project directory.
          . Click "Next". Set "New project name:" to "eclipse_project".
          . Set "Name:" to "eclipse_project".
         . Click "Finish" button. Then a project name "eclipse_project" is created in Eclipse. Simultanesouly, a sub-directory "eclipse_project" is created under directory "nns-1.0".
         . Window "Debug Configurations" will pop up.
Step 3. Set arguments as the configuration file
          . Select "Arguments" tab.
          . Type in the configuration file name "default.cfg" under "Program arguments".
          . Ensure "Use default" is checked.
Step 4. Make the source codes visible to Eclipse by setting "Source" tab as "Path Mapping".
          . Select "Source" tab. You will see "Dafault" under "Source Lookup
            Path:".
          . Click "Add" buttong.
          . Select "Path Mapping" & hit "OK" button.
          . Select "Path Mapping: New Mapping" & hit "Edit" button.
            Then "Path Mappings" window pops up.
            If not, select "Path Mapping:New Mapping" in "Source Lookup Path:"
            and then click "Edit" button on the right.
          . Hit "Add" button. Then in the "Path Mappings" window,
            a new window "Modify the path mappings" may pop up.
          . Hit the "Browse button" for "Local file system path:" & specify
            the project directory "nns-1.0/source_codes".
          . Copy the same path to "Compilation path:" and hit "OK" buttons.
            (I'm not sure if this step is necessary, but the "OK" button is
            not highlighted without doing this.
Step 5. Click the "Apply" button on the right bottom. You may see a message
          "Create Launch Configuration" on the right bottom of the Eclipse
          main window. Now the setting is completed
Step 6. Click "Debug" button to start debugging. You should see the source codes.
             If not, change the perspective to "C/C++". Use "Debug" tab to navigate the source codces.

5.4. How to use/debug with Eclipse

If all the steps so far are successful, the only thing left is to use Eclipse for debugging.
Eclipse is an IDE, so one can edit, navigate, and recompile the source codes.

Step 1. Launch Eclipse.
Step 2. Click the "Run" menu and then "Debug". A shortcut is the F11 key.

Refer to the Eclipse manual to learn how to use the defailed operation.

5.5. Trouble shooting

If workspace is not properly set, nns behaves weirdly. Even if Eclipse can start debugging, it stops in Class Configuration with the following error.

    Error: Cannot open "default.cfg"
              Is the path to "default.cfg" correct?

This is true even if the absolute path to the config file is given. The reason why it behaves weirdly may be because the Eclipse project directory is created in a separate directory
under the same name of the workspace. Be careful because this bug may be not easy to find.

   . "Launching nns-1.2"has encoutered a problem Failed to set program
      arguments,  environment or working directory.
