6.1. Overview

  gnuplot is a plotting tool..

6.2. How to install gnuplot

gnuplot is not a standard command in Ubuntu Linux 8.4, so it should be installed manually. Refer to Ubuntu Linux manual.

6.3. Running gnuplot

When gnuplot is run successfully, the following message is printed by gnuplot, which is stored in an output file.

    GNU Plot is successfully run.

For example, the following is default.out that runs gnuplot successfully.
       ApiGnuplot::configFileName_ = gnuplot_default_config.gpc
       ApiGnuplot::outputFileName_ = gnuplot_default_output.ps
       ApiGnuplot::systemCommandRunGnuplot_ = 'gnuplot gnuplot_default_config.gpc'
       ApiGnuplot::systemCommandRunGnugv_ = 'gv gnuplot_test_output.ps'
       GNU Plot is successfully run
       Use a command gv to check a postscript file (*.ps).

Note a command gv (Ghost Viewer) is also run by the simulator. In other words, the simulator runs the following two system commands.

       $ gnuplot gnuplot_default_config.gp
       $ gv gnuplot_default_output.ps

The system commands are displayed in the output file (default.out).
   
      systemCommandRunGnuplot_ = 'gnuplot gnuplot_default_config.gp'
      systemCommandRunGnugv_ = 'gv gnuplot_default_output.ps'
   
Error message
When gnuplot fails, an error message is generated in an output file. For example, default.out may include the following error message by gnuplot.
       Non-zero value (256) is returned.
       Possible causes for this error include:
           1. GNU Plot is not installed,
           2. GNU Plot configuration file name (*.gp) is incorrect,
           3. GNU Plot execution failed.
