7.1. Convention for function names

An underscore "_" is used for a function related to a variable. For example,

class my_class
{
public:
  void set_inputs_(); Not setInputs_();
  int get_inputs_(); Not getInputs();

private:
int inputs_;
};

The reason for this is to get better "grep" results when the source codes are searched.
Note grep is case sensetive and using the -i option (ignore case) may result in too many other unrelated lines.
For example,

  $ grep "inputs_" *.*

does not show a line "int getInputs_();", but shows "int get_inputs_();".

According to "C++ Programming Style Guidelines" by Geotechnical Software Services at http://geosoft.no/development/cppsty,
1. Any violocation to the guide is allowed if it enhanced readability.
2. The rules can be violoated if there are strong personal objections against them.
